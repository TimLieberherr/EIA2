Aufgabe 5 Antworten

Was beschreibt das Interface AssocStringString und wozu kann es verwendet werden?

Das Interface AssocStringString ist ein assoziatives Array. Dem Array wird "key" als Key vom typ string mitgegeben.

Was geschieht in Zeile 9 und was ist das Ergebnis?

Die Variable "port" wird erstellt vom typ: number. Dieser Variable wird nun zugewiesen, dass Sie in dem Interface "process", der Variable "env" einen Wert für den "PORT" zuweisen soll. In Zeile 10 wird dem Interface den Wert von "Port" zugewiesen wenn die Variable "undefiened" ist. In Zeile 11 wird der Variable "port" der Wert 8100 zugewiesen und somit auch dem Interface im Dokument node.d.ts.

Was geschieht in Zeile 25 von Server Test.ts?

Die Variable "query" wird erstellt vom typ: AssocStringString. Dieser wird zugewiesen das durch "Url.parse" der URL-String in ein Objekt umgewandelt und zurückgegebn wird. Wenn dies "true" ist, dann wird query zu einem neuen Objekt vom typ AssocStringString. Oder anderst gesagt query ist der key von dem Interfacetyp: AssocStringString. "_request.url." wandelt in der Zeile dabei die URL um.

Wie arbeitet die for-in-Schleife in Zeile 29?

Die For-Schleife greift auf "Key" im Objekt "query" zu. In der nächsten Zeile wird wird dann in der Konsole der Wert von "Key" zugehörig zum Objekt "query" ausgegeben.

Was bewirkt die Header-Information Access-Control-Allow-Origin?

Frägt ab ob es der Quelle antworten darf. Sprich es erlaubt die Kommunikation mit dem Browser bzw dem Client der auf den Server zugreifen will.
